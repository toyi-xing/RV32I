# v5.1 Quartus 编译与下载流程

本文档说明 `project_v5.1_fpga` 的 Quartus 工程结构、编译步骤、SOF 下载流程，以及更换 C 程序后的重新打包流程。

## 工程入口

| 项目 | 路径/值 |
| --- | --- |
| Quartus 工程目录 | `project_v5.1_fpga/fpga/quartus/` |
| 工程文件 | `e10_rv32i.qpf` |
| Assignment 文件 | `e10_rv32i.qsf` |
| 时序约束 | `e10_rv32i.sdc` |
| 顶层模块 | `e10_rv32i_top` |
| 目标器件 | `EP4CE10F17C8` |
| 输出目录 | `fpga/quartus/output_files/` |

`e10_rv32i.qsf` 已列出 RTL 文件、目标器件、顶层模块、引脚分配和 IO standard。

## 板级连接

| 信号 | FPGA 引脚 | 说明 |
| --- | --- | --- |
| `sys_clk` | `E1` | 50 MHz 板载时钟。 |
| `sys_rst_n` | `M1` | 低有效复位。 |
| `led[0]` | `A4` | GPIO0 输出。 |
| `led[1]` | `B5` | GPIO0 输出。 |
| `key[0]` | `E16` | GPIO0 输入。 |
| `key[1]` | `E15` | GPIO0 输入。 |
| `uart_rxd` | `B13` | UART RXD。 |
| `uart_txd` | `A13` | UART TXD。 |

所有已分配 IO 当前使用 `3.3-V LVTTL`。

## 时序约束

`e10_rv32i.sdc` 当前只约束板载主时钟：

```tcl
create_clock -name sys_clk -period 20.000 [get_ports {sys_clk}]
derive_clock_uncertainty
```

因此当前签核目标是 50 MHz。若后续加入 PLL、跨时钟域外设或更高频时钟，需要补充对应 generated clock、false path 或 CDC 约束。

## 编译流程

GUI 流程（推荐）：

1. 打开 Quartus Prime Lite。
2. 打开工程 `fpga/quartus/e10_rv32i.qpf`。
3. 确认顶层为 `e10_rv32i_top`，器件为 `EP4CE10F17C8`。
4. 确认要 `fpga\mem` 下的 `current_dmem.mif`、`current_imem.mem` 为要运行的 c 程序编译结果。
5. 运行 `Processing -> Start Compilation`。
6. 编译完成后检查 `output_files/` 下的 `.sof` 和报告文件。

CLI 流程可以在 `fpga/quartus/` 目录运行：

```bash
quartus_sh --flow compile e10_rv32i
```

## 下载流程

编译成功后，使用 Quartus Programmer 下载：

1. 连接 USB-Blaster。
2. 打开 `Tools -> Programmer`。
3. Hardware Setup 选择 USB-Blaster。
4. 添加 `output_files/e10_rv32i.sof`。
5. 勾选 Program/Configure。
6. 点击 Start。

`.sof` 是 SRAM 配置文件，断电后会丢失。若需要上电自启动，需要另行准备配置 Flash 下载流程。

## 更换 C 程序后的流程

更换上板程序时，不能只重新运行 C 编译脚本，还必须重新编译 Quartus，让新的 memory initialization 被打包进 `.sof`。

以新 C 程序为 `project_v5.1_fpga\sw\c\0506_xxxx.c` 为例，顺序为：

```bash
# 编译 C 程序
cd project_v5.1_fpga
scripts/update_fpga_mem.sh 0504
# 编译 FPGA 工程（将上述 C 程序烧录到 IMEM 中）
cd fpga/quartus
quartus_sh --flow compile e10_rv32i
```

然后重新下载 FPGA 工程：

```text
output_files/e10_rv32i.sof
```

流程关系如下：

| 操作 | 是否改变 `.sof` | 说明 |
| --- | --- | --- |
| 修改 `sw/c/*.c` | 否 | 只是软件程序源码变化。 |
| 运行 `scripts/update_fpga_mem.sh` | 否 | 只编译 C 程序并覆盖 `fpga/mem/current_*`。 |
| 重新 Quartus compile | 是 | 把 `current_imem.mem` 和 `current_dmem.mif` 打包进新 `.sof`。 |
| Programmer 下载 `.sof` | 板上生效 | FPGA 重新配置后运行新程序。 |

## 常见检查点

| 现象 | 优先检查 |
| --- | --- |
| 编译找不到 memory init 文件 | `fpga/mem/current_imem.mem` 和 `fpga/mem/current_dmem.mif` 是否存在，路径是否仍相对 Quartus 工程目录有效。 |
| LED 无反应 | 是否下载了最新 `.sof`，`sys_rst_n` 是否释放，`0500` 是否已重新打包。 |
| UART 无输出 | `UART_STATUS_TX_READY` 是否被软件轮询，串口线序和 115200 8N1 设置是否正确。 |
| TIMER polling 正常但 interrupt 异常 | 优先回看 DMEM wrapper 时序、trap entry 栈保存/恢复和 `0504` 测试。 |
| 修改 C 后板上仍运行旧程序 | 是否忘记重新 Quartus compile，或者 Programmer 下载的不是新生成的 `.sof`。 |
