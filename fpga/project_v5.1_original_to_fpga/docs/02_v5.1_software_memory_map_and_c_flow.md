# v5.1 软件地址图与 C 镜像流程

本文档说明 `project_v5.1_fpga` 中软件侧需要遵守的地址图、裸机 C 程序布局，以及如何把 C 程序生成 FPGA 可用的 IMEM/DMEM 初始化文件。

## 地址图

FPGA 版保持 SoC 的 MMIO 地址图不变，只缩小 IMEM/DMEM 容量。

| 区域 | 基地址 | 大小 | 用途 |
| --- | ---: | ---: | --- |
| IMEM | `0x0000_0000` | 16 KiB | 指令存储器，放置 reset stub、trap entry 和 `.text`。 |
| DMEM | `0x0004_0000` | 16 KiB | 数据存储器，放置测试状态字、只读数据、已初始化数据、`.bss` 和栈。 |
| MMIO | `0x0008_0000` | 64 KiB | GPIO、TIMER0、UART0 等外设寄存器窗口。 |

软件常量集中在 `sw/include/platform.h`，RTL 常量集中在 `rtl/common/core_pkg.sv`。修改地址或容量时，这两个文件和链接脚本必须一起检查。

## MMIO 外设

| 外设 | 基地址 | 主要用途 |
| --- | ---: | --- |
| GPIO0 | `0x0008_0000` | LED 输出、KEY 输入、GPIO 中断 pending/status。 |
| TIMER0 | `0x0008_1000` | 32-bit `mtime/mtimecmp`，用于 polling 或 machine timer interrupt。 |
| UART0 | `0x0008_2000` | TXDATA、STATUS、CTRL、RXDATA、RX IRQ pending。 |

UART0 在 FPGA 版中接入真实 UART PHY，软件发送字符前应轮询 `UART_STATUS_TX_READY`。`platform.h` 里的 `uart_putc()` 已经封装了这个等待逻辑。

## C 程序内存布局

链接脚本为 `sw/linker/c_baremetal.ld`。

| 项目 | 地址/范围 | 说明 |
| --- | --- | --- |
| reset entry | `IMEM + 0x0000` | `_start` 位于 `.text.init`，复位后先跳到真正 C 入口。 |
| trap entry | `IMEM + 0x0080` | `.text.trap` 对齐 `core_pkg::MTVEC_RESET`。 |
| C text | trap entry 之后 | `main()` 和普通 C 函数位于 `.text`。 |
| DMEM 保留区 | `DMEM + 0x0000` 到 `DMEM + 0x01ff` | 留给测试状态字和定向测试数据。 |
| `TEST_STATUS_ADDR` | `DMEM + 0x0100` | `crt0.S` 根据 `main()` 返回值写 PASS/FAIL。 |
| C data 起点 | `DMEM + 0x0200` | `.rodata`、`.data` 从这里开始。 |
| stack | DMEM 顶部向下 | `__stack_top = DMEM + 16 KiB`，当前预留 `0x2000` 字节。 |

`crt0.S` 做四件事：

- 设置 `sp` 到 `__stack_top`。
- 清零 `.bss`。
- 调用 `main()`。
- 根据 `main()` 返回值写 `TEST_STATUS_ADDR`，然后停在无限循环。

如果 C 程序定义了 `__trap_handler_c()`，trap entry 会把 `mcause/mepc/mtval` 传给它，并用它的返回值更新 `mepc` 后执行 `mret`。

## 镜像生成流程

FPGA 版使用两个活动初始化文件：

| 文件 | 用途 |
| --- | --- |
| `fpga/mem/current_imem.mem` | IMEM 的 `$readmemh` 初始化文件。 |
| `fpga/mem/current_dmem.mif` | DMEM `altsyncram` 的 MIF 初始化文件。 |

推荐从 `project_v5.1_fpga/` 目录运行：

```bash
scripts/update_fpga_mem.sh 0504
```

参数可以是四位编号、测试名前缀或 `.c` 文件路径。例如：

```bash
scripts/update_fpga_mem.sh 0500
scripts/update_fpga_mem.sh 0504_fpga_timer_irq
scripts/update_fpga_mem.sh sw/c/0505_fpga_key_led_uart.c
```

脚本链路如下：

| 步骤 | 脚本/工具 | 输出 |
| --- | --- | --- |
| 编译链接 | `riscv64-unknown-elf-gcc` | `build/fpga_c/<test>.elf`、`.map` |
| 反汇编 | `riscv64-unknown-elf-objdump` | `build/fpga_c/<test>.dump` |
| 拆分段 | `objcopy -j .text*`、`objcopy -j .dmem_image` | IMEM/DMEM binary |
| 转 IMEM | `scripts/bin2mem32.py` | `<test>_imem.mem` |
| 转 DMEM | `scripts/bin2mem32.py` + `scripts/mem32_to_mif.py` | `<test>_dmem.mif` |
| 覆盖活动镜像 | `scripts/update_fpga_mem.sh` | `fpga/mem/current_imem.mem`、`fpga/mem/current_dmem.mif` |

`build_rv32i_c_image.sh` 会检查 IMEM/DMEM binary 是否超过 16 KiB。超过容量会直接报错，不会更新活动镜像。

## 工具链变量

默认工具链前缀为：

```bash
RISCV_PREFIX=riscv64-unknown-elf
```

可按需要覆盖：

| 变量 | 用途 |
| --- | --- |
| `RISCV_PREFIX` | 改变 `gcc/objcopy/objdump` 的统一前缀。 |
| `RISCV_CC` | 单独指定 C 编译器。 |
| `RISCV_OBJCOPY` | 单独指定 objcopy。 |
| `RISCV_OBJDUMP` | 单独指定 objdump。 |
| `TOOLCHAIN_ENV` | 在编译前 source 一个工具链环境脚本。 |
| `BUILD_DIR` | 改变中间产物输出目录。 |
| `EXTRA_CFLAGS` | 追加 C 编译选项，例如板级极性宏。 |
