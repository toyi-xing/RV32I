# v5.1 已知边界与后续方向

本文档记录 v5.1 FPGA 版当前已经验证的范围、仍需注意的限制，以及后续可扩展方向。它不是缺陷清单，而是工程交接时应显式写下的边界。

## 当前完成范围

| 类别 | 状态 |
| --- | --- |
| CPU core | RV32I core RTL 保持原样，未因 FPGA 上板修改流水线实现。 |
| SoC 地址图 | GPIO、UART、TIMER0 MMIO 基地址保持不变。 |
| FPGA top | 已接入 E10 板载 50 MHz 时钟、复位、2 个 LED、2 个 KEY、UART RX/TX。 |
| IMEM/DMEM | 已替换为 FPGA 可综合/可初始化的 16 KiB + 16 KiB 存储器。 |
| C 裸机流程 | 已支持 C 程序编译、镜像生成、MIF/MEM 更新和 SOF 打包。 |
| 板级验证 | 已通过 LED、KEY、TIMER polling、TIMER interrupt 和综合 demo。 |
| 时序 | 当前 50 MHz 约束下 setup/hold 均通过。 |

## 已知边界

| 边界 | 当前状态 | 影响 |
| --- | --- | --- |
| 工作频率 | 以 50 MHz 为正式目标 | README 中约 52.65 MHz 是粗算理论值，不代表可直接提高时钟。 |
| 存储容量 | IMEM/DMEM 各 16 KiB | C 程序、静态数据、`.bss` 和栈都必须控制在该范围内。 |
| 配置方式 | 当前使用 `.sof` SRAM 下载 | FPGA 断电后程序消失，尚未整理配置 Flash 固化流程。 |
| UART | 简单 TX/RX PHY，无 FIFO | 适合低速状态输出和简单交互，高吞吐场景可能丢数据或需要软件等待。 |
| GPIO | 只实际接出 2 个 KEY 和 2 个 LED | SoC 内部 GPIO 位宽更大，但当前板级 top 只暴露少量 IO。 |
| 中断 | 已验证 TIMER interrupt | GPIO/UART interrupt 相关硬件路径存在，但板级覆盖还不完整。 |
| 时钟域 | 单 `sys_clk` 设计 | 尚未处理多时钟域 CDC 问题。 |
| 调试接口 | 无 JTAG debug、无片上逻辑分析集成说明 | 调试主要依赖 LED、UART、测试程序和 Quartus 报告。 |
| 外部存储 | 未接 SDRAM/SRAM/Flash 数据总线 | 当前程序必须完全放入片上 RAM 初始化镜像。 |

## 非通用假设

这些假设对当前工程成立，但后续复用时需要重新确认：

- 目标器件固定为 `EP4CE10F17C8`，memory wrapper 使用 Cyclone IV E/M9K 相关实现。
- `fpga_dmem.sv` 使用 `altsyncram`，不是跨 FPGA 厂商通用 RAM wrapper。
- IMEM/DMEM 初始化路径相对 Quartus 工程目录成立，移动工程目录时要重新检查路径。
- `sys_clk` 为 50 MHz，UART PHY 参数按 50 MHz / 115200 baud 设置。
- KEY/LED 的有效电平按当前 E10 板卡连接处理，移植到其他板卡时要检查极性。
- C 裸机程序不使用标准库运行时，不依赖操作系统、堆、文件系统或动态内存。

## 需要优先保护的回归点

后续改 RTL 或脚本时，建议把下面几项当作最小回归：

| 优先级 | 回归项 | 原因 |
| ---: | --- | --- |
| 1 | `0500_fpga_led_spark.c` | 快速确认 reset、IMEM、GPIO output 和基础执行。 |
| 2 | `0503_fpga_timer_blink.c` | 确认 TIMER0 MMIO 读写和 polling 路径。 |
| 3 | `0504_fpga_timer_irq.c` | 覆盖 trap、CSR、中断和 DMEM 栈保存/恢复，是 memory timing 关键用例。 |
| 4 | `0505_fpga_key_led_uart.c` | 覆盖多个外设组合路径。 |
| 5 | Quartus STA | 确认 setup/hold 未因改动退化。 |

## 后续扩展方向

| 方向 | 建议 |
| --- | --- |
| UART FIFO | 在 `mmio_uart` 或 PHY 边界加入 TX/RX FIFO，减少软件轮询压力。 |
| GPIO 中断上板验证 | 增加 KEY 触发 GPIO interrupt 的独立 demo，覆盖 MEIP 路径。 |
| UART RX 中断验证 | 增加串口输入触发 UART interrupt 的 demo，覆盖 RXDATA/IRQ_PENDING 行为。 |
| 配置 Flash 固化 | 补充 E10 板卡对应的 Flash 下载和上电自启动流程。 |
| 更大程序空间 | 评估外部存储或更大 FPGA 器件；在 EP4CE10 上继续扩大片上 RAM 会明显挤压资源。 |
| 构建自动化 | 增加一键 `update_fpga_mem + quartus compile` 脚本，减少手工漏步骤。 |
| 报告归档 | 每次阶段性修改后归档 `.fit.rpt`、`.sta.rpt`、资源表和验证程序版本。 |
| 仿真/上板一致性 | 给 FPGA memory wrapper 增加可仿真的替代模型，减少只在板上暴露的问题。 |

## 交接建议

若后续有人接着做 v5.2 或移植到其他板卡，建议先确认三件事：

1. 是否仍要求 CPU core RTL 不动。
2. 目标 FPGA 的 block RAM 读时序是否与当前 wrapper 假设一致。
3. 软件镜像容量、链接脚本和 RTL 地址常量是否同步。

只要这三点没有漂移，当前 v5.1 的文档和脚本就可以作为后续版本的稳定基线。
