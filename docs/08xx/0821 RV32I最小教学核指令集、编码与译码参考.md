# 0821 RV32I 最小教学核指令集、编码与译码参考

> 文档编号：0821  
> 所属系列：082x RISC-V 最小教学核项目实践  
> 文档定位：第一版 RV32I 教学核的指令、编码、立即数和译码控制参考  
> 前置文档：`0801 RISC-V ISA基础.md`、`0802 RISC-V五级流水线与Hazard.md`、`0820 RISC-V最小教学核设计流程与方案.md`

本文不是完整 RISC-V 规范替代品，而是为了写第一版 RV32I 五级流水线教学核时查表使用。目标是让实现者能快速回答：

1. 第一版到底支持哪些指令。
2. 每条指令属于哪种 type。
3. opcode、funct3、funct7 怎么匹配。
4. 哪些指令读 `rs1/rs2`，哪些指令写 `rd`。
5. 译码后应该产生哪些主要控制信号。

本文默认项目假设：

| 项目 | 假设 |
|---|---|
| XLEN | 32 bit |
| 指令长度 | 32 bit，不支持 C 压缩指令 |
| 指令集目标 | RV32I 主线计算、访存、控制流指令优先 |
| 第一版系统指令 | `FENCE` 可先当 NOP；`ECALL/EBREAK` 可暂不支持或作为仿真停止约定 |
| CSR | 第一版不支持 CSR 指令，后续结合 `0803` 扩展 |
| 异常 | 第一版可只产生 `illegal_instr` 调试信号，不做完整 precise trap |

## 第1章 支持范围

### 1.1 第一版建议支持的主线指令

第一版建议优先实现下表中的 RV32I 主线指令。它们已经覆盖五级流水线教学核最关键的数据通路和 hazard 场景。

| 类别 | 指令 | 覆盖能力 |
|---|---|---|
| U-type | `LUI`、`AUIPC` | 大立即数、`PC + imm`、写回选择 |
| I-type ALU | `ADDI`、`SLTI`、`SLTIU`、`XORI`、`ORI`、`ANDI`、`SLLI`、`SRLI`、`SRAI` | 立即数、比较、逻辑、移位 |
| R-type ALU | `ADD`、`SUB`、`SLT`、`SLTU`、`XOR`、`OR`、`AND`、`SLL`、`SRL`、`SRA` | 双寄存器 ALU、forwarding 主路径 |
| load | `LB`、`LH`、`LW`、`LBU`、`LHU` | 地址计算、访存、符号/零扩展、load-use |
| store | `SB`、`SH`、`SW` | 地址计算、byte enable、store data forwarding |
| branch | `BEQ`、`BNE`、`BLT`、`BGE`、`BLTU`、`BGEU` | 控制相关、signed/unsigned 比较、flush |
| jump | `JAL`、`JALR` | 无条件跳转、`PC+4` 写回、寄存器间接跳转 |

### 1.2 第一版可暂缓的指令

完整 RV32I 还涉及同步/系统类指令。第一版可以先做简化处理，但文档和代码里必须说清楚，不要声称“完整实现 RV32I”。

| 指令/类别 | 编码位置 | 第一版建议 |
|---|---|---|
| `FENCE` | `opcode=0001111`，`funct3=000` | 可先当 NOP，不改变架构状态 |
| `ECALL` | `opcode=1110011`，`funct3=000`，`imm12=000000000000` | 可暂不支持；或仿真中约定为停止/报错 |
| `EBREAK` | `opcode=1110011`，`funct3=000`，`imm12=000000000001` | 可暂不支持；或仿真中约定为停止/断点 |
| CSR 指令 | `opcode=1110011`，`funct3 != 000` | 第一版不支持，后续进入 CSR/trap 扩展 |

如果第一版没有完整 trap 机制，那么遇到未支持指令时比较实用的做法是：

- decoder 拉高 `illegal_instr_o`。
- testbench 看到该信号后报错停止。
- 后续做 `0803` 相关扩展时，再把非法指令接入 precise exception。

## 第2章 指令格式与立即数

### 2.1 32 bit 指令公共字段

RISC-V 32 bit 指令最低 7 bit 是 opcode。常见字段位置如下。图上方的数字表示指令本身的 bit 边界，例如 `31 25` 表示 `instr[31:25]`；框内的 `imm[20]`、`imm[11]` 表示立即数拼回去之后的 bit 编号，不是十进制数值。为了避免单 bit 字段过窄，图中的列宽只保留大致宽窄关系，不按精确位宽比例缩放。

```text
R-type

 31       25 24     20 19     15 14    12 11     7 6          0
+-----------+---------+---------+--------+--------+------------+
|   funct7  |   rs2   |   rs1   | funct3 |   rd   |   opcode   |
+-----------+---------+---------+--------+--------+------------+

I-type

 31             20 19     15 14    12 11     7 6          0
+-----------------+---------+--------+--------+------------+
|    imm[11:0]    |   rs1   | funct3 |   rd   |   opcode   |
+-----------------+---------+--------+--------+------------+

S-type

 31         25 24     20 19     15 14    12 11       7 6          0
+-------------+---------+---------+--------+----------+------------+
|  imm[11:5]  |   rs2   |   rs1   | funct3 | imm[4:0] |   opcode   |
+-------------+---------+---------+--------+----------+------------+

B-type

 31   31 30          25 24     20 19     15 14    12 11       8 7     7 6          0
+-------+--------------+---------+---------+--------+----------+-------+------------+
|imm[12]|  imm[10:5]   |   rs2   |   rs1   | funct3 | imm[4:1] |imm[11]|   opcode   |
+-------+--------------+---------+---------+--------+----------+-------+------------+

U-type

 31                       12 11     7 6          0
+---------------------------+--------+------------+
|         imm[31:12]        |   rd   |   opcode   |
+---------------------------+--------+------------+

J-type

 31   31 30                  21 20   20 19           12 11     7 6          0
+-------+----------------------+-------+---------------+--------+------------+
|imm[20]|      imm[10:1]       |imm[11]|   imm[19:12]  |   rd   |   opcode   |
+-------+----------------------+-------+---------------+--------+------------+
```

### 2.2 立即数生成规则

建议在 RTL 中单独写 `imm_gen`。这样 decoder 只决定 `imm_sel`，立即数拼接集中在一个模块里，便于测试。

| imm 类型 | 用途 | 生成规则 |
|---|---|---|
| I-imm | I-type ALU、load、JALR | `$signed({instr[31:20]})` 符号扩展到 32 bit |
| S-imm | store | `$signed({instr[31:25], instr[11:7]})` 符号扩展到 32 bit |
| B-imm | branch offset | `$signed({instr[31], instr[7], instr[30:25], instr[11:8], 1'b0})` 符号扩展到 32 bit |
| U-imm | LUI、AUIPC | `{instr[31:12], 12'b0}` |
| J-imm | JAL offset | `$signed({instr[31], instr[19:12], instr[20], instr[30:21], 1'b0})` 符号扩展到 32 bit |

用 LaTeX 写成更紧凑的形式：

$$
\text{imm}_I = \text{sign\_ext}(\text{inst}[31:20])
$$

$$
\text{imm}_S = \text{sign\_ext}(\{\text{inst}[31:25], \text{inst}[11:7]\})
$$

$$
\text{imm}_B = \text{sign\_ext}(\{\text{inst}[31], \text{inst}[7], \text{inst}[30:25], \text{inst}[11:8], 1'b0\})
$$

$$
\text{imm}_U = \{\text{inst}[31:12], 12'b0\}
$$

$$
\text{imm}_J = \text{sign\_ext}(\{\text{inst}[31], \text{inst}[19:12], \text{inst}[20], \text{inst}[30:21], 1'b0\})
$$

注意：如果 `imm_gen` 已经把 B/J offset 的最低位补成 `0`，后面计算 branch/JAL target 时就不要再额外左移一次。也就是说：

$$
\text{branch\_target} = \text{pc} + \text{imm}_B
$$

$$
\text{jal\_target} = \text{pc} + \text{imm}_J
$$

`JALR` 不使用 B/J 这种拼接方式，而是 I-type 立即数：

$$
\text{jalr\_target} = (\text{rs1} + \text{imm}_I) \mathbin{\&} \sim 32'h1
$$

最低位清 0 是 ISA 对 `JALR` 目标地址的要求。

## 第3章 opcode 速查

| 指令类别 | opcode 二进制 | opcode 十六进制 | 说明 |
|---|---:|---:|---|
| `LOAD` | `0000011` | `0x03` | `LB/LH/LW/LBU/LHU` |
| `MISC-MEM` | `0001111` | `0x0f` | `FENCE` 等 |
| `OP-IMM` | `0010011` | `0x13` | I-type ALU |
| `AUIPC` | `0010111` | `0x17` | `rd = PC + immU` |
| `STORE` | `0100011` | `0x23` | `SB/SH/SW` |
| `OP` | `0110011` | `0x33` | R-type ALU |
| `LUI` | `0110111` | `0x37` | `rd = immU` |
| `BRANCH` | `1100011` | `0x63` | 条件分支 |
| `JALR` | `1100111` | `0x67` | 寄存器间接跳转 |
| `JAL` | `1101111` | `0x6f` | PC 相对跳转 |
| `SYSTEM` | `1110011` | `0x73` | `ECALL/EBREAK/CSR` |

第一版 decoder 可以先用 opcode 进入大类，再用 `funct3/funct7` 区分具体指令。

## 第4章 指令编码与行为表

### 4.1 U-type

```text
 31                       12 11     7 6          0
+---------------------------+--------+------------+
|         imm[31:12]        |   rd   |   opcode   |
+---------------------------+--------+------------+
```

| 指令 | 汇编格式 | opcode | 行为 | 控制要点 |
|---|---|---|---|---|
| `LUI` | `lui rd, imm20` | `0110111` | `rd = immU` | 不读 `rs1/rs2`，写回来源为 U-imm |
| `AUIPC` | `auipc rd, imm20` | `0010111` | `rd = PC + immU` | 不读 `rs1/rs2`，ALU A 选 PC，B 选 U-imm |

`LUI` 和 `AUIPC` 的指令字段中没有真正的 `rs1/rs2`。hazard 检测不能因为这些 bit 恰好像寄存器编号，就误认为它们读寄存器。

### 4.2 Jump

```text
JAL：J-type

 31   31 30                  21 20   20 19           12 11     7 6          0
+-------+----------------------+-------+---------------+--------+------------+
|imm[20]|      imm[10:1]       |imm[11]|   imm[19:12]  |   rd   |   opcode   |
+-------+----------------------+-------+---------------+--------+------------+

JALR：I-type

 31             20 19     15 14    12 11     7 6          0
+-----------------+---------+--------+--------+------------+
|    imm[11:0]    |   rs1   | funct3 |   rd   |   opcode   |
+-----------------+---------+--------+--------+------------+
```

| 指令 | type | 汇编格式 | opcode | funct3 | 行为 | 控制要点 |
|---|---|---|---|---|---|---|
| `JAL` | J-type | `jal rd, offset` | `1101111` | 无 | `rd = PC + 4; next_pc = PC + immJ` | 不读 `rs1/rs2`，写回 `PC+4`，产生 redirect |
| `JALR` | I-type | `jalr rd, imm(rs1)` | `1100111` | `000` | `rd = PC + 4; next_pc = (rs1 + immI) & ~1` | 读 `rs1`，可能需要 forwarding，写回 `PC+4` |

`JALR` 常用于函数返回，例如伪指令 `ret` 通常是 `jalr x0, 0(x1)`。因此它不是“随便跳一下”，而是函数调用/返回链路的重要指令。

### 4.3 Branch

```text
 31   31 30          25 24     20 19     15 14    12 11       8 7     7 6          0
+-------+--------------+---------+---------+--------+----------+-------+------------+
|imm[12]|  imm[10:5]   |   rs2   |   rs1   | funct3 | imm[4:1] |imm[11]|   opcode   |
+-------+--------------+---------+---------+--------+----------+-------+------------+
```

| 指令 | 汇编格式 | opcode | funct3 | 比较方式 | taken 条件 | 控制要点 |
|---|---|---|---|---|---|---|
| `BEQ` | `beq rs1, rs2, offset` | `1100011` | `000` | 相等比较 | `rs1 == rs2` | 读 `rs1/rs2`，taken 时 `PC = PC + immB` |
| `BNE` | `bne rs1, rs2, offset` | `1100011` | `001` | 不等比较 | `rs1 != rs2` | 同上 |
| `BLT` | `blt rs1, rs2, offset` | `1100011` | `100` | 有符号小于 | `$signed(rs1) < $signed(rs2)` | signed 比较 |
| `BGE` | `bge rs1, rs2, offset` | `1100011` | `101` | 有符号大于等于 | `$signed(rs1) >= $signed(rs2)` | signed 比较 |
| `BLTU` | `bltu rs1, rs2, offset` | `1100011` | `110` | 无符号小于 | `rs1 < rs2` | unsigned 比较 |
| `BGEU` | `bgeu rs1, rs2, offset` | `1100011` | `111` | 无符号大于等于 | `rs1 >= rs2` | unsigned 比较 |

branch 不写 `rd`，但会改变 PC。branch 比较操作数来自 `rs1/rs2`，所以它同样需要 forwarding。例如：

```asm
add  x1, x2, x3
beq  x1, x0, target
```

如果 `beq` 在 EX 阶段比较，就必须拿到 `add` 的最新结果，否则会用旧的 `x1` 判断跳不跳。

### 4.4 Load

```text
 31             20 19     15 14    12 11     7 6          0
+-----------------+---------+--------+--------+------------+
|    imm[11:0]    |   rs1   | funct3 |   rd   |   opcode   |
+-----------------+---------+--------+--------+------------+
```

| 指令 | 汇编格式 | opcode | funct3 | 访存宽度 | 写回行为 |
|---|---|---|---|---|---|
| `LB` | `lb rd, imm(rs1)` | `0000011` | `000` | 8 bit | 读 byte，符号扩展到 32 bit 写 `rd` |
| `LH` | `lh rd, imm(rs1)` | `0000011` | `001` | 16 bit | 读 halfword，符号扩展到 32 bit 写 `rd` |
| `LW` | `lw rd, imm(rs1)` | `0000011` | `010` | 32 bit | 读 word 写 `rd` |
| `LBU` | `lbu rd, imm(rs1)` | `0000011` | `100` | 8 bit | 读 byte，零扩展到 32 bit 写 `rd` |
| `LHU` | `lhu rd, imm(rs1)` | `0000011` | `101` | 16 bit | 读 halfword，零扩展到 32 bit 写 `rd` |

load 地址：

$$
\text{addr} = \text{rs1} + \text{imm}_I
$$

第一版可以先假设访问地址合法、memory 总是响应。若要支持不对齐访问异常、访问 fault、MMIO wait state，就进入后续扩展范围。

### 4.5 Store

```text
 31         25 24     20 19     15 14    12 11       7 6          0
+-------------+---------+---------+--------+----------+------------+
|  imm[11:5]  |   rs2   |   rs1   | funct3 | imm[4:0] |   opcode   |
+-------------+---------+---------+--------+----------+------------+
```

| 指令 | 汇编格式 | opcode | funct3 | 写入宽度 | 行为 |
|---|---|---|---|---|---|
| `SB` | `sb rs2, imm(rs1)` | `0100011` | `000` | 8 bit | 把 `rs2[7:0]` 写入 `rs1 + immS` 指向的 byte lane |
| `SH` | `sh rs2, imm(rs1)` | `0100011` | `001` | 16 bit | 把 `rs2[15:0]` 写入对应 halfword lane |
| `SW` | `sw rs2, imm(rs1)` | `0100011` | `010` | 32 bit | 把 `rs2[31:0]` 写入 word |

store 地址：

$$
\text{addr} = \text{rs1} + \text{imm}_S
$$

load/store 都有地址对齐问题。RISC-V 的访存地址不是由指令编码天然保证对齐，而是运行时由 `rs1 + imm` 算出来；编译器、链接脚本和正常 C 程序通常会生成对齐访问，但手写汇编、错误指针或刻意构造的测试仍然可能产生不对齐地址。第一版教学核可以先不支持不对齐访问，但处理原则应是“检测并报错/停止”，而不是把地址低位强行清零。

| 访问类型 | 涉及指令 | 对齐要求 | 第一版建议 |
|---|---|---|---|
| byte 访问 | `LB/LBU/SB` | 无要求，任意 byte 地址都可以访问 | 正常访问，用 `addr[1:0]` 选择 byte lane |
| halfword 访问 | `LH/LHU/SH` | `addr[0] == 1'b0` | 若不满足，报 misaligned load/store，不真正读写 |
| word 访问 | `LW/SW` | `addr[1:0] == 2'b00` | 若不满足，报 misaligned load/store，不真正读写 |

因此，硬件不要这样做：

```systemverilog
addr = {addr[31:2], 2'b00}; // 错误：这会把程序给出的地址偷偷改成另一个地址
```

更合理的是在 MEM 阶段根据访存宽度检查 `addr` 低位。若当前教学核还没有 trap/CSR，就可以在 testbench 中 `halt/error`；若后续实现异常机制，再升级为 `load address misaligned` 或 `store/AMO address misaligned`，并禁止错误指令产生副作用：store 不能真的写 memory，load 不能写回 `rd`。

store 的 `rs1` 是地址基址，`rs2` 是要写入 memory 的数据。两者都可能需要 forwarding，尤其是：

```asm
add  x3, x1, x2
sw   x3, 0(x4)
```

这里 `sw` 的 store data 来自刚刚计算出的 `x3`，不能只考虑 ALU 操作数 forwarding。

### 4.6 I-type ALU

```text
 31             20 19     15 14    12 11     7 6          0
+-----------------+---------+--------+--------+------------+
|    imm[11:0]    |   rs1   | funct3 |   rd   |   opcode   |
+-----------------+---------+--------+--------+------------+
```

| 指令 | 汇编格式 | opcode | funct3 | funct7/imm[11:5] | 行为 |
|---|---|---|---|---|---|
| `ADDI` | `addi rd, rs1, imm` | `0010011` | `000` | 任意 | `rd = rs1 + immI` |
| `SLTI` | `slti rd, rs1, imm` | `0010011` | `010` | 任意 | `rd = ($signed(rs1) < $signed(immI)) ? 1 : 0` |
| `SLTIU` | `sltiu rd, rs1, imm` | `0010011` | `011` | 任意 | `rd = (rs1 < unsigned(sign_ext(immI))) ? 1 : 0` |
| `XORI` | `xori rd, rs1, imm` | `0010011` | `100` | 任意 | `rd = rs1 ^ immI` |
| `ORI` | `ori rd, rs1, imm` | `0010011` | `110` | 任意 | `rd = rs1 \| immI` |
| `ANDI` | `andi rd, rs1, imm` | `0010011` | `111` | 任意 | `rd = rs1 & immI` |
| `SLLI` | `slli rd, rs1, shamt` | `0010011` | `001` | `0000000` | `rd = rs1 << shamt` |
| `SRLI` | `srli rd, rs1, shamt` | `0010011` | `101` | `0000000` | `rd = rs1 >> shamt`，逻辑右移 |
| `SRAI` | `srai rd, rs1, shamt` | `0010011` | `101` | `0100000` | `rd = $signed(rs1) >>> shamt`，算术右移 |

RV32I shift immediate 只使用 5 bit shamt，即 `instr[24:20]`。`SLLI/SRLI/SRAI` 不是普通 12 bit 立即数运算，decoder 需要检查 `instr[31:25]`。

`SLTIU` 容易写错：它的立即数仍然先按 I-type 规则符号扩展到 32 bit，然后再把两个操作数当 unsigned 值比较。也就是说，`sltiu rd, rs1, -1` 中的立即数会变成 `32'hffff_ffff`，不是 `12'hfff`。

### 4.7 R-type ALU

```text
 31       25 24     20 19     15 14    12 11     7 6          0
+-----------+---------+---------+--------+--------+------------+
|   funct7  |   rs2   |   rs1   | funct3 |   rd   |   opcode   |
+-----------+---------+---------+--------+--------+------------+
```

| 指令 | 汇编格式 | opcode | funct3 | funct7 | 行为 |
|---|---|---|---|---|---|
| `ADD` | `add rd, rs1, rs2` | `0110011` | `000` | `0000000` | `rd = rs1 + rs2` |
| `SUB` | `sub rd, rs1, rs2` | `0110011` | `000` | `0100000` | `rd = rs1 - rs2` |
| `SLL` | `sll rd, rs1, rs2` | `0110011` | `001` | `0000000` | `rd = rs1 << rs2[4:0]` |
| `SLT` | `slt rd, rs1, rs2` | `0110011` | `010` | `0000000` | `rd = ($signed(rs1) < $signed(rs2)) ? 1 : 0` |
| `SLTU` | `sltu rd, rs1, rs2` | `0110011` | `011` | `0000000` | `rd = (rs1 < rs2) ? 1 : 0` |
| `XOR` | `xor rd, rs1, rs2` | `0110011` | `100` | `0000000` | `rd = rs1 ^ rs2` |
| `SRL` | `srl rd, rs1, rs2` | `0110011` | `101` | `0000000` | `rd = rs1 >> rs2[4:0]` |
| `SRA` | `sra rd, rs1, rs2` | `0110011` | `101` | `0100000` | `rd = $signed(rs1) >>> rs2[4:0]` |
| `OR` | `or rd, rs1, rs2` | `0110011` | `110` | `0000000` | `rd = rs1 | rs2` |
| `AND` | `and rd, rs1, rs2` | `0110011` | `111` | `0000000` | `rd = rs1 & rs2` |

R-type 是 forwarding 测试的主力。特别注意同一个 `rd` 被连续写时，EX/MEM 前递优先级应高于 MEM/WB，避免拿到更旧的值。

### 4.8 同步/系统类指令

```text
FENCE：MISC-MEM 类，字段位置类似 I-type，但 imm[11:0] 被拆成 fm/pred/succ

 31    28 27    24 23    20 19     15 14    12 11     7 6          0
+--------+--------+--------+---------+--------+--------+------------+
|   fm   |  pred  |  succ  |   rs1   | funct3 |   rd   |   opcode   |
+--------+--------+--------+---------+--------+--------+------------+

ECALL/EBREAK：SYSTEM 类

 31             20 19     15 14    12 11     7 6          0
+-----------------+---------+--------+--------+------------+
|    imm[11:0]    |   rs1   | funct3 |   rd   |   opcode   |
+-----------------+---------+--------+--------+------------+
```

| 指令 | 汇编格式 | opcode | funct3 | imm12 | 第一版建议 |
|---|---|---|---|---|---|
| `FENCE` | `fence pred, succ` | `0001111` | `000` | `fm/pred/succ` | 可当 NOP |
| `ECALL` | `ecall` | `1110011` | `000` | `000000000000` | 可报不支持，或仿真停止 |
| `EBREAK` | `ebreak` | `1110011` | `000` | `000000000001` | 可报不支持，或仿真断点 |

`FENCE` 真正涉及 memory ordering。单 hart、无 cache、无乱序、无 MMIO reorder 的第一版教学核里，通常可以把它当成 NOP。`ECALL/EBREAK` 若要做成规范行为，就需要 trap、CSR、`mepc/mcause/mtvec`，这属于 `0803` 和后续扩展。

## 第5章 译码控制信号建议

### 5.1 推荐控制信号

第一版 decoder 可以输出下面这些控制信号。具体命名可在 `0822` 统一，但含义建议保持稳定。

| 控制信号 | 典型取值 | 作用 |
|---|---|---|
| `illegal_instr` | `0/1` | 当前指令是否未支持或编码非法 |
| `uses_rs1` | `0/1` | hazard unit 是否需要检查 `rs1` 相关 |
| `uses_rs2` | `0/1` | hazard unit 是否需要检查 `rs2` 相关 |
| `reg_we` | `0/1` | 指令是否写 `rd`，最终 WB 仍要屏蔽 `rd=x0` |
| `imm_sel` | `I/S/B/U/J/NONE` | 选择立即数生成方式 |
| `op_a_sel` | `RS1/PC/ZERO` | ALU A 输入来源 |
| `op_b_sel` | `RS2/IMM/FOUR` | ALU B 输入来源 |
| `alu_op` | `ADD/SUB/SLT/...` | ALU 运算类型 |
| `branch_op` | `NONE/BEQ/BNE/...` | branch 比较类型 |
| `jump` | `0/1` | 是否为 `JAL/JALR` |
| `jalr` | `0/1` | 是否使用 `rs1 + immI` 作为跳转基址 |
| `mem_re` | `0/1` | 是否 load |
| `mem_we` | `0/1` | 是否 store |
| `mem_size` | `B/H/W` | 访存宽度 |
| `mem_unsigned` | `0/1` | load 是否零扩展 |
| `wb_sel` | `ALU/MEM/PC4/IMM` | 写回数据来源 |

### 5.2 大类控制表

| 指令类别 | `uses_rs1` | `uses_rs2` | `reg_we` | 写回来源 | `imm_sel` | `op_a_sel` | `op_b_sel` | `wb_sel` | memory | PC 控制 |
|---|---:|---:|---:|---|---|---|---|---|---|---|
| `LUI` | 0 | 0 | 1 | IMM | U | ZERO 或 IMM 特判 | IMM | IMM/ALU | 无 | `PC+4` |
| `AUIPC` | 0 | 0 | 1 | ALU | U | PC | IMM | ALU | 无 | `PC+4` |
| I-type ALU | 1 | 0 | 1 | ALU | I | RS1 | IMM | ALU | 无 | `PC+4` |
| R-type ALU | 1 | 1 | 1 | ALU | NONE | RS1 | RS2 | ALU | 无 | `PC+4` |
| load | 1 | 0 | 1 | MEM | I | RS1 | IMM | MEM | read | `PC+4` |
| store | 1 | 1 | 0 | - | S | RS1 | IMM | 无 | write | `PC+4` |
| branch | 1 | 1 | 0 | - | B | PC | IMM(ALU负责计算地址，比较用`branch_unit`) | 无 | 无 | taken 时 `PC+immB` |
| `JAL` | 0 | 0 | 1 | PC4 | J | PC | IMM | PC4 | 无 | `PC+immJ` |
| `JALR` | 1 | 0 | 1 | PC4 | I | RS1 | IMM | PC4 | 无 | `(rs1+immI)&~1` |
| `FENCE` | 0 | 0 | 0 | - | NONE | 无 | 无 | 无 | 无 | `PC+4` |
| `ECALL/EBREAK` | 0 | 0 | 0 | - | NONE | 无 | 无 | 无 | 无 | 第一版可报错/停止 |

`uses_rs1/uses_rs2` 很重要。它们不等于“指令编码里是否存在这些 bit”。例如 `LUI` 的 bit `[19:15]` 不是 `rs1`，如果 hazard unit 把它当 `rs1`，会产生假相关和不必要 stall。

### 5.3 `alu_op` 译码查表

5.2 的大类控制表回答“这条指令大概要走哪条数据通路”，本小节回答更具体的问题：ALU 到底要做哪一种运算。写 RTL 时通常先用 `opcode` 分大类，再用 `funct3/funct7` 分小类，最后产生 `alu_op`。

第一版建议至少定义下面这些 `alu_op`：

| `alu_op` | ALU 行为 | 典型用途 |
|---|---|---|
| `ALU_ADD` | `rs1 + rs2`、`rs1 + immI`、`rs1 + immS`、`pc + immB/immJ/immU` | 加法、立即数加法、load/store 地址、`AUIPC/branch/JAL/JALR` target 地址 |
| `ALU_SUB` | `rs1 - rs2` | `SUB` |
| `ALU_SLT` | `$signed(rs1) < $signed(rs2/immI)` | 有符号小于比较，结果写 `0/1` |
| `ALU_SLTU` | `unsigned(rs1) < unsigned(rs2/immI)` | 无符号小于比较，结果写 `0/1` |
| `ALU_XOR` | `rs1 ^ rs2/immI` | 按位异或 |
| `ALU_OR` | `rs1 | rs2/immI` | 按位或 |
| `ALU_AND` | `rs1 & rs2/immI` | 按位与 |
| `ALU_SLL` | `rs1 << shamt` | 逻辑左移 |
| `ALU_SRL` | `rs1 >> shamt` | 逻辑右移，高位补 0 |
| `ALU_SRA` | `$signed(rs1) >>> shamt` | 算术右移，高位补符号位 |
| `ALU_COPY_B` 或 `ALU_PASS_IMM` | 直接输出 B 输入 | 可选，用于 `LUI`；也可以让 WB 直接选择 `immU` 而不经过 ALU |

I-type ALU 和 R-type ALU 的 `alu_op` 可以按下表查：

| 指令 | opcode | funct3 | funct7 或 `instr[31:25]` | `op_a_sel` | `op_b_sel` | `alu_op` | ALU 行为 |
|---|---|---|---|---|---|---|---|
| `ADDI` | `0010011` | `000` | 任意立即数高位 | RS1 | IMM | `ALU_ADD` | `rs1 + immI` |
| `SLTI` | `0010011` | `010` | 任意立即数高位 | RS1 | IMM | `ALU_SLT` | `$signed(rs1) < $signed(immI)` |
| `SLTIU` | `0010011` | `011` | 任意立即数高位 | RS1 | IMM | `ALU_SLTU` | `unsigned(rs1) < unsigned(immI)` |
| `XORI` | `0010011` | `100` | 任意立即数高位 | RS1 | IMM | `ALU_XOR` | `rs1 ^ immI` |
| `ORI` | `0010011` | `110` | 任意立即数高位 | RS1 | IMM | `ALU_OR` | `rs1 | immI` |
| `ANDI` | `0010011` | `111` | 任意立即数高位 | RS1 | IMM | `ALU_AND` | `rs1 & immI` |
| `SLLI` | `0010011` | `001` | `0000000` | RS1 | IMM | `ALU_SLL` | `rs1 << instr[24:20]` |
| `SRLI` | `0010011` | `101` | `0000000` | RS1 | IMM | `ALU_SRL` | `rs1 >> instr[24:20]` |
| `SRAI` | `0010011` | `101` | `0100000` | RS1 | IMM | `ALU_SRA` | `$signed(rs1) >>> instr[24:20]` |
| `ADD` | `0110011` | `000` | `0000000` | RS1 | RS2 | `ALU_ADD` | `rs1 + rs2` |
| `SUB` | `0110011` | `000` | `0100000` | RS1 | RS2 | `ALU_SUB` | `rs1 - rs2` |
| `SLL` | `0110011` | `001` | `0000000` | RS1 | RS2 | `ALU_SLL` | `rs1 << rs2[4:0]` |
| `SLT` | `0110011` | `010` | `0000000` | RS1 | RS2 | `ALU_SLT` | `$signed(rs1) < $signed(rs2)` |
| `SLTU` | `0110011` | `011` | `0000000` | RS1 | RS2 | `ALU_SLTU` | `unsigned(rs1) < unsigned(rs2)` |
| `XOR` | `0110011` | `100` | `0000000` | RS1 | RS2 | `ALU_XOR` | `rs1 ^ rs2` |
| `SRL` | `0110011` | `101` | `0000000` | RS1 | RS2 | `ALU_SRL` | `rs1 >> rs2[4:0]` |
| `SRA` | `0110011` | `101` | `0100000` | RS1 | RS2 | `ALU_SRA` | `$signed(rs1) >>> rs2[4:0]` |
| `OR` | `0110011` | `110` | `0000000` | RS1 | RS2 | `ALU_OR` | `rs1 | rs2` |
| `AND` | `0110011` | `111` | `0000000` | RS1 | RS2 | `ALU_AND` | `rs1 & rs2` |

非普通 ALU 指令也会复用 ALU，常见查表如下：

| 指令/场景 | opcode | funct3 | `op_a_sel` | `op_b_sel` | `alu_op` | ALU 输出含义 |
|---|---|---|---|---|---|---|
| `LUI` | `0110111` | 无 | ZERO | IMM | `ALU_COPY_B` 可选 | `immU`；也可不进 ALU，WB 直接选 `immU` |
| `AUIPC` | `0010111` | 无 | PC | IMM | `ALU_ADD` | `pc + immU`，写回 `rd` |
| `LB/LH/LW/LBU/LHU` 地址 | `0000011` | `000/001/010/100/101` | RS1 | IMM | `ALU_ADD` | load 地址 `rs1 + immI` |
| `SB/SH/SW` 地址 | `0100011` | `000/001/010` | RS1 | IMM | `ALU_ADD` | store 地址 `rs1 + immS` |
| branch target | `1100011` | `000/001/100/101/110/111` | PC | IMM | `ALU_ADD` | `pc + immB`，作为 taken 时的 next PC |
| `JAL` target | `1101111` | 无 | PC | IMM | `ALU_ADD` | `pc + immJ`，无条件跳转 target |
| `JALR` target 基础值 | `1100111` | `000` | RS1 | IMM | `ALU_ADD` | `rs1 + immI`，之后 target 清 bit0 |

在这个教学核方案里，branch 和 jump 的地址计算统一交给 ALU：branch 计算 `pc + immB`，`JAL` 计算 `pc + immJ`，`JALR` 计算 `rs1 + immI` 后再清 bit0。也就是说，ALU 输出的是“候选跳转地址”或“实际跳转地址”，不是 branch 是否成立的判断结果。

branch 是否 taken 由单独的 `branch_unit` 判断。`branch_unit` 输入通常是 forwarded 之后的 `rs1/rs2` 和 `branch_op`，输出 `branch_taken`：

| branch 指令 | opcode | funct3 | `branch_op` | `branch_unit` 判断 |
|---|---|---|---|---|
| `BEQ` | `1100011` | `000` | `BR_EQ` | `rs1 == rs2` |
| `BNE` | `1100011` | `001` | `BR_NE` | `rs1 != rs2` |
| `BLT` | `1100011` | `100` | `BR_LT` | `$signed(rs1) < $signed(rs2)` |
| `BGE` | `1100011` | `101` | `BR_GE` | `$signed(rs1) >= $signed(rs2)` |
| `BLTU` | `1100011` | `110` | `BR_LTU` | `unsigned(rs1) < unsigned(rs2)` |
| `BGEU` | `1100011` | `111` | `BR_GEU` | `unsigned(rs1) >= unsigned(rs2)` |

最后的 PC 选择可以理解成：

```text
branch : next_pc = branch_taken ? alu_target : pc + 4
JAL    : next_pc = alu_target
JALR   : next_pc = {alu_target[31:1], 1'b0}
其他   : next_pc = pc + 4
```

这里有两个容易写错的点：

1. 对 `ADDI/SLTI/SLTIU/XORI/ORI/ANDI` 这类非 shift I-type 指令，`instr[31:20]` 整体是立即数，`instr[31:25]` 不是功能码，不应该拿它去匹配 `0000000`。
2. 对 `SLLI/SRLI/SRAI`，`instr[31:25]` 才需要当作编码合法性检查；实际移位量是 `instr[24:20]`，也就是 RV32I 的 5 bit `shamt`。

### 5.4 写回与 x0

decoder 可以对 `ADD x0, x1, x2` 这类指令仍然输出 `reg_we=1`，但真正写 GPR 时必须加 `rd != 0`：

```systemverilog
assign gpr_we = wb_valid && wb_reg_we && (wb_rd != 5'd0);
```

hazard/forwarding 匹配后级目的寄存器时，也必须排除 `rd=x0`：

```systemverilog
assign exmem_match_rs1 =
    exmem_valid && exmem_reg_we && (exmem_rd != 5'd0) &&
    idex_uses_rs1 && (exmem_rd == idex_rs1);
```

否则会把“写 x0 的指令”误认为真的产生了新值，导致 forwarding 选择错误。

## 第6章 decoder 写法骨架

下面是教学核 decoder 的参考写法骨架，不要求逐字照搬。关键是默认值完整、非法编码能落到 `illegal_instr`，并且每条指令的 `uses_rs1/uses_rs2/reg_we/wb_sel` 清楚。

```systemverilog
always_comb begin
    dec = '0;
    dec.illegal_instr = 1'b1;

    unique case (opcode)
        OPCODE_LUI: begin
            dec.illegal_instr = 1'b0;
            dec.reg_we        = 1'b1;
            dec.imm_sel       = IMM_U;
            dec.wb_sel        = WB_IMM;
        end

        OPCODE_AUIPC: begin
            dec.illegal_instr = 1'b0;
            dec.reg_we        = 1'b1;
            dec.imm_sel       = IMM_U;
            dec.op_a_sel      = OP_A_PC;
            dec.op_b_sel      = OP_B_IMM;
            dec.alu_op        = ALU_ADD;
            dec.wb_sel        = WB_ALU;
        end

        OPCODE_OP_IMM: begin
            dec.uses_rs1      = 1'b1;
            dec.reg_we        = 1'b1;
            dec.imm_sel       = IMM_I;
            dec.op_a_sel      = OP_A_RS1;
            dec.op_b_sel      = OP_B_IMM;
            dec.wb_sel        = WB_ALU;

            unique case (funct3)
                3'b000: begin dec.illegal_instr = 1'b0; dec.alu_op = ALU_ADD; end // ADDI
                3'b010: begin dec.illegal_instr = 1'b0; dec.alu_op = ALU_SLT; end // SLTI
                3'b011: begin dec.illegal_instr = 1'b0; dec.alu_op = ALU_SLTU; end // SLTIU
                3'b100: begin dec.illegal_instr = 1'b0; dec.alu_op = ALU_XOR; end // XORI
                3'b110: begin dec.illegal_instr = 1'b0; dec.alu_op = ALU_OR;  end // ORI
                3'b111: begin dec.illegal_instr = 1'b0; dec.alu_op = ALU_AND; end // ANDI
                3'b001: begin
                    if (funct7 == 7'b0000000) begin
                        dec.illegal_instr = 1'b0;
                        dec.alu_op        = ALU_SLL;
                    end
                end
                3'b101: begin
                    if (funct7 == 7'b0000000) begin
                        dec.illegal_instr = 1'b0;
                        dec.alu_op        = ALU_SRL;
                    end else if (funct7 == 7'b0100000) begin
                        dec.illegal_instr = 1'b0;
                        dec.alu_op        = ALU_SRA;
                    end
                end
                default: ;
            endcase
        end

        default: begin
            dec.illegal_instr = 1'b1;
        end
    endcase
end
```

实际工程里还要继续补 `OP`、`LOAD`、`STORE`、`BRANCH`、`JAL`、`JALR` 等大类。建议不要把所有控制逻辑都塞进一个几百行的 `always_comb` 后就不再整理；可以先用表格定义，再写 RTL。

## 第7章 测试清单

实现 `0821` 对应译码后，至少写这些 directed test：

| 测试 | 目的 |
|---|---|
| 每个 opcode 至少一条 | 确认大类译码正确 |
| 每个 funct3/funct7 组合 | 确认 ALU、load/store、branch 子类型正确 |
| shift immediate 非法 funct7 | 确认 `SLLI/SRLI/SRAI` 编码检查有效 |
| `rd=x0` | 确认写回被屏蔽，forwarding 不从 x0 匹配 |
| `LUI/AUIPC/JAL` | 确认不读 `rs1/rs2`，不会产生假 hazard |
| branch 正负 offset | 确认 B-imm 拼接和符号扩展正确 |
| JAL 正负 offset | 确认 J-imm 拼接和符号扩展正确 |
| JALR odd target | 确认目标地址 bit0 被清 0 |
| load 扩展 | `LB/LH/LBU/LHU` 符号/零扩展正确 |
| store byte enable | `SB/SH/SW` 写入 lane 正确 |
| unsupported instruction | 确认 `illegal_instr` 行为明确 |
